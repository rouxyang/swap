Swap Web 应用开发框架入门指南

感谢使用 Swap Web 应用开发框架。本文以从零开始开发一个 Blog 来介绍如何使用 Swap 开发 Web 应用。

一、建立 Web 应用工作目录

假设想在 /path/to/myapp 目录下存放 Web 应用程序并在该目录下进行开发。只要把解压出来的框架目录（通常是 swap-x.y.z）里的 app/ 目录下的所有文件和子目录复制到该目录下即可。复制完成后，请确认该目录下的目录结构为如下结构：

  • config/
  • controller/
  • data/
  • doc/
  • filter/
  • helper/
  • task/
  • logic/
  • kern/
  • vendor/
  • utility/
  • view/
  • web/
  • path.php

二、Web 应用运行环境安装与配置

Swap 可以运行在 Nginx + PHP-FPM (Linux/Unix) 和 Apache 2.2 + mod_php (Linux/Unix/Windows) 环境上，目前还没测试在其它运行环境上是否也能正常工作，因此建议使用这两种运行环境之一来运行 Swap。

Swap 基于 PHP 5.3，因此运行环境的 PHP 版本号需要 >= 5.3.0。核心 Swap 对 PHP 扩展的要求如下：

  • Core
  • date
  • iconv
  • json
  • pcre
  • Reflection
  • SPL
  • standard

通常默认编译的 PHP 都包含了这些扩展。如果你喜欢手动编译 PHP，而且喜欢 --disable-all，那么请务必添加 --with-iconv --enable-json 以启用 iconv 和 json 扩展。
